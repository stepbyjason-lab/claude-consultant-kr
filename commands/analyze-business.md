# 사업 전략 분석 실행

현재 프로젝트의 코드베이스를 직독하여 한국 시장 기준 사업 전략 리포트를 생성합니다.

## 실행 절차

1. **리포트 디렉토리 준비**
   - `./reports/` 디렉토리가 없으면 생성

2. **consultant-kr 에이전트 호출**
   - 분석 초점: `$ARGUMENTS` (지정되지 않으면 "전반적 사업 전략")
   - 에이전트에게 다음을 지시:
     - 현 디렉토리 코드베이스를 Read/Grep/Glob으로 직독
     - README, package.json, 주요 설정 파일을 우선 확인
     - 공우산(사실→해석→제안) 프레임워크로 분석 수행
     - 한국 시장 컨텍스트(국내 경쟁사·결제·플랫폼) 반영
     - 출력 포맷 8단계 전체 구조 준수

3. **리포트 저장**
   - 파일명: `./reports/analysis-{YYYYMMDD-HHMM}.md`
   - 파일 상단에 메타데이터 블록 추가:
     ```yaml
     ---
     생성일시: {현재 KST}
     분석 초점: {$ARGUMENTS 또는 "전반적 사업 전략"}
     분석 대상: {프로젝트 루트 경로}
     ---
     ```

4. **요약 출력**
   - 저장 완료 메시지와 파일 경로
   - 논점 정의(1번 섹션) 원문
   - 추천 시책(6번 섹션) 핵심 3줄

## 실행 제약

- 리포트 저장 전 반드시 `mkdir -p ./reports` 로 디렉토리 보장
- 파일 시각은 KST 기준 (`date +%Y%m%d-%H%M`)
- consultant-kr 에이전트가 호출되지 않으면 즉시 중단하고 `/agents`로 설치 상태 확인 안내
