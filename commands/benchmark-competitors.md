# 경쟁사 벤치마킹 분석

현 프로젝트를 국내 주력 경쟁사와 비교하여 차별화 포인트를 도출합니다.

## 실행 절차

1. **자사 현황 파악**
   - 코드베이스 구조 직독
   - 구현된 기능 / 미구현 기능 목록화
   - 기술 스택 및 인프라 구성 확인

2. **경쟁사 매핑**
   - 카테고리: `$ARGUMENTS`
   - 에이전트 정의에 포함된 "경쟁사 벤치마크 매핑" 테이블을 기준으로 국내 주력 플레이어 3~5곳 선정
   - 필요 시 WebSearch로 최신 가격/기능 정보 확인
   - **글로벌 플레이어는 참고용으로만** 사용 (Wix, Shopify 등)

3. **consultant-kr 에이전트 호출**
   - 분석 프레임워크: **3C + SWOT 병행**
     - 3C: Customer / Competitor / Company
     - SWOT: 내부(S/W) × 외부(O/T)
   - 출력 요구사항:
     - 기능 매트릭스 비교표 (자사 vs 경쟁사 N곳)
     - 가격 포지셔닝 차트 (텍스트 기반 2축 매핑)
     - **차별화 가능한 포인트 TOP 3** (우선순위 포함)
     - **따라잡아야 할 약점 TOP 3** (우선순위 포함)
     - 국내 시장 특유의 진입 장벽 (네이버·카카오 생태계 연동 등)

4. **추천 액션**
   - 공우산 프레임워크로 정리
   - Phase 1(즉시 실행) / Phase 2(3개월) / Phase 3(6개월 이후) 구분
   - 각 액션의 **예상 리소스**(개발 공수, 마케팅 예산 수준) 명기

## 출력 형식

```markdown
## 3C 분석

### Customer (고객)
- 타깃 세그먼트 정의
- 핵심 니즈 TOP 3

### Competitor (경쟁)
| 업체 | 강점 | 약점 | 가격대 | 차별점 |
| --- | --- | --- | --- | --- |

### Company (자사)
- 현재 포지션
- 보유 자산과 제약

## SWOT 매트릭스

|   | 기회(O) | 위협(T) |
| --- | --- | --- |
| 강점(S) | SO 전략 | ST 전략 |
| 약점(W) | WO 전략 | WT 전략 |

## 추천 액션 (우선순위 순)
...
```

## 주의사항

- 경쟁사 정보는 **공개 자료(홈페이지, 블로그, 뉴스)** 범위로 한정
- 추측으로 경쟁사의 내부 지표(MAU, 매출)를 단정하지 말 것
- 불확실한 정보는 "추정" 표시

## 한국어 윤문 (humanize-korean 스킬 활용 — 옵트인)

위 결과를 사용자에게 출력한 직후, 다음 경로 중 하나에 `SKILL.md` 가 존재하는지 확인한다:
- `~/.claude/skills/humanize-korean/SKILL.md`
- `./.claude/skills/humanize-korean/SKILL.md`

**존재하면**: `humanize-korean` 스킬을 호출하여 위 결과의 한국어 서술 부분에만 윤문을 적용한 뒤, "🪶 한국어 윤문본" 섹션으로 추가 출력한다. 원본 결과는 그대로 유지.

**존재하지 않으면**: 단계 건너뛰고 마지막에 한 줄 안내만:
- `💡 한국어 윤문 추가: npx consultant-kr-cli humanize --local`

**윤문 시 절대 변경 금지**: 경쟁사명/제품명, 표 데이터, SWOT 매트릭스, 3C 항목 헤더, "추정" 표기. 자연어 설명 문장만 다듬는다.
